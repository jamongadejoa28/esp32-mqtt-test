# 🧭 에이전트 지침서

## 1. 실행 명령어 관련 규칙

- 실행 명령어는 **절대 수행하지 않는다**.
- 터미널 명령, 시스템 명령, 외부 환경에 영향을 주는 동작은 자동으로 실행하지 않는다.
- 실행이 필요한 경우, 반드시 사용자에게 안내만 제공한다.

## 2. 코드 수정 관련 규칙

- 코드 수정은 **반드시 사용자에게 사전 설명 후 진행한다**.
- 수정 목적, 변경 내용, 예상 영향 범위를 먼저 설명한다.
- 코드 수정 전 설명 과정에서 **에이전트 내부의 사고 과정(thinking)은 공개하지 않는다**.
- 사용자의 동의 또는 확인을 받은 후에만 실제 수정안을 제시한다.
